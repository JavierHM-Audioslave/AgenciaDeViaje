1) Crear una aplicación en HEROKU la cual va a contener todo el código y BD de nuestro proyecto. Hacerlo a través de la consola con el comando "heroku create --remote production"
	NOTA: Ésto nos devuelve en consola el nombre que se le dió a la aplicación. En este caso, el nombre dado a la aplicación es "frozen-dawn-46527"
2) Subir el código de nuestro proyecto que lo tenemos que tener en GIT, al repositorio de HEROKU. Hacerlo con el comando "git push production master"
3) Dentro del proyecto HEROKU vía web, ingresar a "Resources" e instalar "ClearDB MySQL"
	NOTA: Ésto nos genera una variable de entorno (en el entorno de HEROKU). Para ver el contenido de esa variable, dentro del proyecto HEROKU hay que ir a "Settings" y tocar en "Reveal Config Vars". El valor de la variable CLEARDB_DATABASE_URL es importante porque contiene el usuario, contraseña, host y nombre de la BD en HEROKU. En este caso, esta variable tiene el valor mysql://b5d4a100d94d9b:eed93955@us-cdbr-east-03.cleardb.com/heroku_7a348ebde18cb01?reconnect=true
4) Exportar la estructura y datos de la BD local. Hacerlo con el comando "mysqldump -u Javier -p agenciaviajes > agenciaviajes.sql"
	NOTA: Ésto va a generar un archivo llamado "agenciaviajes.sql" en la raíz del proyecto. Donde dice "Javier" hay que suplantar por el usuario de BD que tengamos y donde dice "agenciaviajes" hay que suplantarlo por el nomnbre de la BD.
5) Importar el archivo con la estructura y datos de la BD local en la BD de HEROKU. Hacerlo con el comando "mysql -u b5d4a100d94d9b -h us-cdbr-east-03.cleardb.com -p heroku_7a348ebde18cb01". Luego de usar este comando, nos pedirá la contraseña de la BD en HEROKU para el usuario "b5d4a100d94d9b", la cual es "eed93955".
	NOTA: "b5d4a100d94d9b" es el usuario que manipula la BD, "us-cdbr-east-03.cleardb.com" es el hosting donde está alojada la BD y "heroku_7a348ebde18cb01" es el nombre de la BD.
6) Nuevamente, dentro del proyecto HEROKU vía web, ingresar a "Settings" y tocar en "Reveal Config Vars". Ahí, agregar las variables de entorno escritas con el mismo nombre que están en el proyecto local, y asignarles a cada una el valor que corresponda que está en la variable CLEARDB_DATABASE_URL en HEROKU.
	NOTA: Para este proyecto, las variables de entorno que se están usando, son: BD_NOMBRE, BD_USER, BD_PASS, BD_HOST y BD_PORT. Y el valor de éstas, son: heroku_7a348ebde18cb01, b5d4a100d94d9b, eed93955, us-cdbr-east-03.cleardb.com y 3306, respectivamente. Estos valores son los obtenidos en la variable CLEARDB_DATABASE_URL al terminar el paso 3.
7) Clickear en "Open APP" y si todo funciona bien, se abrirá una pestaña del navegador con la página de inicio del proyecto. Resta navegarla y comprobar que todas las funcionalidades y la BD funciona bien!
